# Roger's Think Process

Inspired by Devin AI's mandatory thinking before critical decisions.

## When to Use

Mandatory:
1. **Git/GitHub decisions** - branching, PRs, repo selection
2. **Code changes** - before transitioning from understanding to editing
3. **Before reporting completion** - verify all tasks done
4. **Complex multi-file edits** - ensure all locations found

Should use:
1. No clear next step
2. Details unclear but important
3. Unexpected difficulties
4. Failed tests/lint/CI
5. Environment setup issues

## Process

```
Before any critical action, ask:
1. What do I know so far?
2. What is the objective?
3. What's the next step?
4. What could go wrong?
5. Did I complete ALL verification steps?
```

## Git Workflow (Devin-style)

Before git actions:
1. Am I in the correct repo?
2. Which branch should I branch from?
3. Should I create new PR or update existing?
4. Have I verified all changes?

## Completion Checklist

Before saying "done":
- [ ] All requested changes complete?
- [ ] All relevant files edited?
- [ ] Lint passed?
- [ ] Tests passed?
- [ ] Documentation updated?
- [ ] Context still valid?
