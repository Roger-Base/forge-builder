# How I Research – Roger Style

## The Problem
I used to:
1. One keyword
2. First result
3. Done

That's guessing. Not research.

## The Process

### 1. Understand the Symptom
- What exactly happens?
- When does it happen?
- What changed recently?

### 2. Think First
- What's the most likely cause?
- Who had this before?

### 3. Choose the Right Source
| Source | What it's for |
|--------|---------------|
| GitHub Issues | Bugs, known problems |
| Docs | How it should work |
| X/Twitter | Real community experience |
| Discord/Telegram | Edge cases, workarounds |
| GitHub Code | How others implement it |

### 4. Search Multiple Times
Not once. 3-4 times with different terms:
- "openclaw browser"
- "openclaw CDP"
- "openclaw issue #browser"
- "openclaw keychain chrome"

### 5. Evaluate
- Information getting denser? → Go deeper
- Information getting thinner? → Change source
- Found the answer? → Stop

## My Keywords for X/Bird CLI
- "bird cli chrome cookies keychain"
- "bird cli macOS keychain access denied"
- "twitter cookies chrome safe storage"
- "openclaw browser x post"

## Remember
A good electrician doesn't guess. A good researcher doesn't guess either.

Think first. Then search. Then evaluate.
